📖 Help & Features
💬 Chat & Conversations
Streaming Responses
Responses appear token-by-token in real time via Server-Sent Events. You can stop generation anytime by clicking the stop button.
Conversation History
All conversations are saved locally in your browser's storage. Switch between chats using the sidebar, or create new ones with Ctrl+N.
Retry Responses
Hover over any AI response to reveal action buttons. Click 🔄 Retry to regenerate from that point in the conversation.
Copy Messages
Hover over any message and click 📋 Copy to copy the full text. Code blocks have their own copy buttons.
🧠 Context Window
What is the Context Window?
The context window is the maximum number of tokens (words/subwords) the model can process at once. It includes all messages in the conversation. Click the context indicator below the input to see detailed usage.
Context Health Indicators
● Healthy Under 75% — plenty of room
● Warning 75–90% — consider starting a new chat
● Critical Over 90% — responses may be truncated
⚡ Performance & Hardware
Generation Speed
After each response, generation stats appear showing tokens/second, total tokens, and elapsed time. These appear below the AI's response.
Memory Monitor
The sidebar shows live GPU VRAM and system RAM usage, updated every 5 seconds. Click the refresh icon to update instantly.
OpenVINO Acceleration
Models run locally using Intel's OpenVINO runtime. Check the sidebar to see which device (GPU/CPU) is active.

⚙️ Settings & Configuration
Inference Settings Menu
Click the ⚙️ Settings button in the top header. You can adjust the execution device, max response length, and context window size.
Context Window & Sliders
Adjust max input tokens to prevent GPU memory overflow, and max new tokens to control response lengths. Click Save Settings to apply them dynamically.
🔌 VS Code & MCP Integration
VS Code Extensions (Continue, Cline, Roo Code, etc.)
Connect standard extensions using the OpenAI compatibility layer. Set the Base URL to http://localhost:5000/v1.
Claude Desktop & MCP Servers
Configure custom MCP servers or desktop prompts to query your local model at /api/mcp/chat. Refer to docs/mcp_and_extension_integration.md for config examples.
